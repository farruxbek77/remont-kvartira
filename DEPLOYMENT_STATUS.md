## 📝 Important Notes

### Data Storage
- Uses localStorage (browser-based storage)
- Data does NOT sync across devices
- Each browser has its own data
- Clearing browser data will delete all requests

### Admin Access
- Username: `admin`
- Password: `admin2013`
- Session-based authentication
- Logout clears session

### Limitations (Static Site)
- No real backend
- No database
- No email notifications
- No file uploads to server (images stored as base64 in localStorage)
- Data limited by browser localStorage size (~5-10MB)

### For Production Use
Consider implementing:
- Real backend API (Node.js, Python, etc.)
- Database (PostgreSQL, MongoDB, etc.)
- File storage service (AWS S3, Cloudinary, etc.)
- Email service (SendGrid, Mailgun, etc.)
- Authentication service (Auth0, Firebase Auth, etc.)
